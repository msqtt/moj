## 项目设计

### 代码结构

项目使用六边形架构组织，对于一个服务应用来说，业务逻辑不依赖任何具体的应用代码，而是依赖统一的抽象接口。应用层实现这些抽象接口从而做到**控制反转**，减少业务逻辑与应用代码的耦合性，提高系统的可扩展性。

如图所示：

<center>
    <img src="./assets/hexagonal-architecture.drawio.png"/>
</center>

为了近一步降低微服务间各服务间的代码耦合，各应用要能做到独立部署与更新。

目录结构：
```
.
├── apps
├── builds
├── docs
├── domain
├── go.work
├── go.work.sum
...
```
